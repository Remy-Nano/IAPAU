# Diagrammes UML - Prompt Challenge

> Dernière mise à jour: 2026-03-05
>
> Pour `mermaid.live`, le plus simple est d'ouvrir un des fichiers `.mmd` ci-dessous
> et copier-coller tout le contenu du fichier (sans backticks).

1. Use case: `docs/uml/soutenance/use-case.mmd`
2. Class diagram: `docs/uml/soutenance/class.mmd`
3. Sequence diagram (index): `docs/uml/soutenance/sequence.mmd`
4. Sequence - Etudiant: `docs/uml/soutenance/sequence-etudiant.mmd`
5. Sequence - Admin: `docs/uml/soutenance/sequence-admin.mmd`
6. Sequence - Examinateur: `docs/uml/soutenance/sequence-examinateur.mmd`
7. Activity diagram: `docs/uml/soutenance/activity.mmd`
8. MCD NoSQL (reel): `docs/uml/soutenance/mcd-nosql.mmd`
9. MCD SQL (fictif): `docs/uml/soutenance/mcd-sql.mmd`
10. MLD SQL (fictif): `docs/uml/soutenance/mld-sql.mmd`
11. MPD SQL (fictif): `docs/uml/soutenance/mpd-sql.mmd`
12. MLD SQL cible (normalise): `docs/uml/soutenance/mld-sql-cible.mmd`
13. MPD SQL cible (normalise): `docs/uml/soutenance/mpd-sql-cible.mmd`

## Rappel

- coller un seul diagramme à la fois dans `mermaid.live`
- le code doit toujours commencer par le type du diagramme
  - `flowchart LR` (use case)
  - `classDiagram`
  - `sequenceDiagram`
  - `flowchart TD` (activity)
